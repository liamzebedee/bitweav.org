BitWeav introduction for hackers

This introduction proceeds the laymen introduction on the homepage, and introduces BitWeav in more technical detail.

BitWeav is a work-in-progress implementation of a design for a completely distributed peer-to-peer micropublishing network.

The basis of the network is that every message is published on at least one topic, whether that be a profile feed, a hashtag or a thread. I use a peer-to-peer publish-subscribe overlay network design called PolderCast to achieve this — nodes join other nodes interested in topics and forward messages to eachother in a Ring structure.

BitWeav is the first project of its kind to achieve autonomous discovery of content through topics. Other networks like OStatus rely on the user to input their subscriptions to find new content, but BitWeav achieves this through the publish-subscribe overlay.

Some other cool things that BitWeav employs:

  • A scalable publish-subscribe overlay network design that doesn't require rendez-vous nodes for topics (meaning only people who are interested in a topic will help distribute messages on it)
  • Collaborative filtering of content through various incentives for network nodes.
  • Identicons for visually distinguishing between different users
  • Automatic highlighting of URLs for interoperability with other apps. For example, if a Tent URL is encountered, it will be displayed nicely in a card below the message.
  • Automatic user discovery from website using <link> tag
  • Message fields for the thread of a message and the message it is in reply to

Code

Like Bitcoin, BitWeav is composed of a frontend graphical client backed by a backend daemon. The frontend communications to the backend via a language-agnostic RPC protocol — this is the local API. The backend does most the work, and communicates to peers in the BitWeav network using the Peer API, a binary RPC protocol.

The frontend client is built using node-webkit, a framework for developing web apps on the desktop using NodeJS. It uses AngularJS as its base, with Bootstrap 3 for styling.

The backend client is a daemon built using Ruby. I haven't done much on it yet.
