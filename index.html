Introduction

  • In the last five years Twitter has demonstrated the value of micropublishing as a communications medium.
  • Starting conversations with strangers, spreading ideas for change, and sparking revolution — all in real-time.
  • But Twitter has some significant issues...

Issues with Twitter

  • The Twitter network is closed and centralised. As a result of this lack of openness, users have little control over their data. The security and reliability of the service is impeded by its centralised nature.
  • The service is corporately owned and controlled by Twitter Inc., and thus is subject to influence from governments (PRISM, censorship) and shareholders (more profit).

Alternative?

  • There exists an alternative already called OStatus, an open web protocol for distributed micropublishing. OStatus builds upon a suite of other existing protocols like Atom feeds and PuSH hubs that have become the lingua franca of open content distribution.
  • Let's dive in and see how it works...

OStatus Overview

OStatus starts with the idea that nearly all websites have Atom feeds, so logically we should start by storing messages as a list of entries. Now this allows us to periodically get the new content of a user, but micropublishing isn't all that. It's also the way people interact...

OStatus Overview

So instead of a simple Atom feed, OStatus uses an open format for social activity feeds called ActivityStreams. ActivityStreams allow us to express everything we need to follow someone's activities, however we have to repeatedly pull down the feed, which isn't realtime...

OStatus Overview

Instead of repeatedly pulling new feed updates, we instead register to a webhook and get new updates pushed to us. This protocol is called PubSubHubbub (PuSH), and it allows us to get updates in real-time, but how do we send content back?

OStatus Overview

Finally to send content (ActivityStream entries) to another site, OStatus employs Salmon, a protocol for decentralised commentary.

OStatus in summary

  • OStatus is a collection of standards for federated web micropublishing.
    • Atom feeds for publishing content.
    • PuSH hubs for realtime distribution of new content.
    • Salmon for decentralised commentary.
  • But OStatus still has issues...

OStatus issues

  1. No official implementation (in comparison to Twitter) which makes for a fragmented user experience.
  2. No 'killer app' to stir adoption, just a set of protocols that *could* achieve microblogging
  3. No content discovery through hashtags!

The solution

    BitWeav — a project to bring open distributed micropublishing to the masses:
    1. Official implementation: Official desktop client that connects to a backend daemon (allows for self-hosting or SaaS setup)
    2. Killer app: be the 'indie' Twitter, for open distributed micropublishing. The idea is to do one thing, and to do it well.
    3. Hashtags: P2P publish-subscribe network based on PolderCast for distributing messages on topics.

Status

  • The design is done: I've been working on BitWeav for six months now, iterating through various designs. I wrote a 14 page whitepaper in July that outlined a flavour of BitWeav without the web and just purely P2P. Since then I've moved over to the web as the platform.
  • The implementation is in progress: Both the backend daemon and graphical client are in development on GitHub, and are populated with development tasks. The graphical client is a desktop web app using node-webkit and AngularJS, and the backend daemon is a Ruby gem. I am currently implementing the graphical client, heading around 50%.
